Expand description

Object-safe trait for post-retrieval refinement strategies.

Each implementation encapsulates the algorithm and its parameters. The engine holds a Box<dyn RefinementStrategy> and delegates to it, replacing the previous match on RefinementMode.

fn refine( &self, retriever: &mut PerTokenRetriever, query_key: &[f32], first_stage: Vec<RetrievalResult>, k: usize, owner_filter: Option<OwnerId>, source: Option<&dyn CellSource>, ) -> Vec<RetrievalResult>

Apply refinement to first-stage candidates.

Semantics are identical to apply: re-score or re-query the candidates, returning at most k results.

source: when Some, strategies that need per-cell tokens (mean-centered, whitened, PRF) load them lazily via the CellSource instead of relying on the retriever’s in-memory token store. Engines wired for lazy mode pass Some; legacy callers that still use the in-memory store pass None.
