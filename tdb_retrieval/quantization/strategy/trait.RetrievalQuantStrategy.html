Expand description

Strategy: how the retriever quantizes and dequantizes a single token vector for the scoring path.

Implementations must be deterministic — the same input always yields the same output — and must be Send + Sync because the engine holds them behind a shared reference and dispatches from concurrent query contexts.

fn quantize_token(&self, token: &[f32]) -> QuantizedToken

Quantize one token vector.

Returns QuantizedToken::empty() on input that doesn’t match self.dim() — callers treat this as “skip” rather than failing the surrounding batch.

fn dequantize_token(&self, quantized: &QuantizedToken) -> Vec<f32>

Dequantize one token back to f32.

Returns an empty Vec if quantized.dim does not match self.dim(). The recovered vector has length self.dim().
