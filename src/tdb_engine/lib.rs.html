1//! The brain of `TardigradeDB` — a unified facade that turns five independent
2//! subsystems into a single, coherent memory for autonomous AI agents.
3//!
4//! # Why does this crate exist?
5//!
6//! An LLM agent that can reason over its own history needs four things to happen
7//! on every interaction: **persist** the current KV cache, **retrieve** relevant
8//! past context, **organize** causal relationships between memories, and **govern**
9//! which memories survive over time. Each of these is handled by a separate crate
10//! in `TardigradeDB` — but no agent should need to coordinate them manually.
11//!
12//! `tdb-engine` is the **Facade** that hides that complexity. It exposes two
13//! primary operations — [`mem_write`] and [`mem_read`] — and everything else
14//! (quantization, indexing, importance scoring, tier promotion, WAL logging,
15//! causal edge tracking) happens as automatic side effects.
16//!
17//! # What happens when you write
18//!
19//! A single call to [`mem_write`] triggers a cascade across all four layers:
20//!
21//! ```text
22//! mem_write(owner, layer, key, value, salience, parent)
23//!   │
24//!   ├─ Governance:  ImportanceScorer::new(salience) + on_update(+5)
25//!   │               TierStateMachine::evaluate → Draft / Validated / Core
26//!   │
27//!   ├─ Storage:     BlockPool::append → Q4-compress key+value, fsync to segment
28//!   │               (on-disk metadata includes importance + tier for crash recovery)
29//!   │
30//!   ├─ Retrieval:   Pipeline::insert → PerTokenRetriever + BruteForce
31//!   │               SLB::insert → cache mean-pooled key for hot-path lookups
32//!   │
33//!   ├─ Index:       if Vamana active → insert (graph ANN)
34//!   │               if cell_count ≥ threshold → lazy-activate Vamana
35//!   │
36//!   └─ Trace:       if parent_cell_id provided →
37//!                     WAL::append(AddEdge) → fsync (Observer pattern)
38//!                     TraceGraph::add_edge(child → parent, CausedBy)
39//! ```
40//!
41//! Governance is computed **before** persistence so the on-disk metadata matches
42//! the in-memory state — this is required for correct Memento-pattern recovery
43//! when the engine reopens.
44//!
45//! # What happens when you read
46//!
47//! A single call to [`mem_read`] runs a three-stage retrieval chain
48//! (**Chain of Responsibility** pattern), merging results across stages:
49//!
50//! ```text
51//! mem_read(query_key, k, owner_filter)
52//!   │
53//!   ├─ SLB (hot cache, separate from pipeline)
54//!   │  INT8 quantized mean-pooled keys, NEON SDOT / AVX2
55//!   │  Sub-5μs per query at 4096 entries
56//!   │  Exploits conversational locality — recent cells are cached
57//!   │
58//!   ├─ Pipeline Stage 1: PerTokenRetriever (primary)
59//!   │  Inverted Multi-Key Index with Top5Avg scoring
60//!   │  INT8 per-token dot products, tuneable via PerTokenConfig
61//!   │  100% recall at 100 memories (validated on Qwen3-0.6B)
62//!   │
63//!   ├─ Pipeline Stage 2: BruteForce (fallback)
64//!   │  Exhaustive scaled dot-product: score = (q · k) / √d_k
65//!   │  Exact results, owner-filtered per-agent partitions
66//!   │
67//!   └─ Pipeline Stage 3: Vamana (optional, lazy-activated)
68//!      DiskANN-style graph with robust pruning
69//!      Activated when cell count crosses threshold
70//!
71//!   Then for each result:
72//!     → recency_decay(Δt) multiplier (21-day half-life)
73//!     → BlockPool::get → dequantize full cell from disk
74//!     → ImportanceScorer::on_access(+3) → potential tier promotion
75//!     → SLB::insert → cache for future hot-path hits
76//! ```
77//!
78//! Results are deduplicated by `CellId`, sorted by decay-adjusted score, and
79//! truncated to k.
80//!
81//! # Crash recovery and cross-process sync (Memento pattern)
82//!
83//! On [`Engine::open`] and [`refresh`], the engine rebuilds all in-memory
84//! derived state from durable sources:
85//!
86//! ```text
87//! Engine::open(dir) / Engine::refresh()
88//!   ├─ BlockPool::open/refresh_index → scan segments
89//!   ├─ Pipeline rebuild → fresh PerTokenRetriever + BruteForce
90//!   │  (Vamana reset; re-activated lazily if cell count ≥ threshold)
91//!   ├─ For each persisted cell:
92//!   │    Pipeline::insert(key) + SLB::insert(mean_pooled_key)
93//!   │    ImportanceScorer::new(persisted_importance)
94//!   │    TierStateMachine::with_tier(persisted_tier)
95//!   ├─ WAL::replay → rebuild TraceGraph from logged edges
96//!   ├─ PackDirectory::from_cells → rebuild pack membership
97//!   ├─ DeletionLog::refresh → filter deleted packs
98//!   └─ next_id = max(persisted_ids) + 1
99//! ```
100//!
101//! `refresh()` provides the same guarantee as a fresh `open()`: two Engine
102//! handles sharing a path stay synchronized. The vLLM connector relies on
103//! this for scheduler ↔ worker coordination.
104//!
105//! The engine can crash at any point and recover all state that was durably
106//! committed (fsync'd). In-flight writes that didn't complete fsync are
107//! discarded — partial record detection in segment scanning handles this.
108//!
109//! # Causal memory (Trace graph)
110//!
111//! When a cell is written with `parent_cell_id = Some(parent)`, the engine
112//! records a directed `CausedBy` edge in the Trace graph, durably logged via
113//! the WAL before being applied in memory. This enables causal chain queries:
114//!
115//! ```text
116//! trace_ancestors(cell_c) → [cell_b, cell_a]
117//! ```
118//!
119//! If cell C was caused by B, and B was caused by A, the engine traverses
120//! the causal graph transitively.
121//!
122//! # Synaptic bank (`LoRA` adapters)
123//!
124//! Beyond episodic KV memory, the engine also persists per-agent **`LoRA` adapters**
125//! via [`store_synapsis`] / [`load_synapsis`]. These encode stable preferences and
126//! patterns as low-rank weight deltas applied to the base model at runtime —
127//! the "slow memory" complement to the fast episodic KV cache.
128//!
129//! # Python bridge
130//!
131//! The `tdb-python` crate wraps this engine via `PyO3`. From Python:
132//!
133//! ```python
134//! import tardigrade_db as tdb
135//! engine = tdb.Engine("/tmp/tardigrade")
136//! cell_id = engine.mem_write(owner=42, layer=12, key=np_key, value=np_val, salience=80.0, parent_cell_id=None)
137//! results = engine.mem_read(query_key=np_query, k=5, owner=None)
138//! ancestors = engine.trace_ancestors(cell_id)
139//! ```
140//!
141//! # Usage (Rust)
142//!
143//! ```rust,no_run
144//! use tdb_engine::engine::Engine;
145//!
146//! let dir = std::path::Path::new("/tmp/tardigrade");
147//! let mut engine = Engine::open(dir).unwrap();
148//!
149//! // Capture a KV pair from transformer layer 12.
150//! let cell_id = engine.mem_write(
151//!     /* owner */ 42,
152//!     /* layer */ 12,
153//!     /* key */ &[0.1f32; 128],
154//!     /* value */ vec![0.2f32; 128],
155//!     /* salience */ 60.0,
156//!     /* parent */ None,
157//! ).unwrap();
158//!
159//! // Retrieve the most relevant cells for a query key.
160//! let results = engine.mem_read(&[0.1f32; 128], /* k */ 5, None).unwrap();
161//! if let Some(top) = results.first() {
162//!     println!("cell {} score={:.3} tier={:?}", top.cell.id, top.score, top.tier);
163//! }
164//!
165//! // Query causal ancestors.
166//! let ancestors = engine.trace_ancestors(cell_id);
167//!
168//! // Close and reopen — all state is recovered from disk.
169//! drop(engine);
170//! let engine2 = Engine::open(dir).unwrap();
171//! assert_eq!(engine2.cell_count(), 1);
172//! ```
