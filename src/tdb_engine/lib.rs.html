1//! The brain of `TardigradeDB` — a unified facade that turns five independent
2//! subsystems into a single, coherent memory for autonomous AI agents.
3//!
4//! # Why does this crate exist?
5//!
6//! An LLM agent that can reason over its own history needs four things to happen
7//! on every interaction: **persist** the current KV cache, **retrieve** relevant
8//! past context, **organize** causal relationships between memories, and **govern**
9//! which memories survive over time. Each of these is handled by a separate crate
10//! in `TardigradeDB` — but no agent should need to coordinate them manually.
11//!
12//! `tdb-engine` is the **Facade** that hides that complexity. It exposes two
13//! primary operations — [`mem_write`] and [`mem_read`] — and everything else
14//! (quantization, indexing, importance scoring, tier promotion, WAL logging,
15//! causal edge tracking) happens as automatic side effects.
16//!
17//! # What happens when you write
18//!
19//! A single call to [`mem_write`] triggers a cascade across all four layers:
20//!
21//! ```text
22//! mem_write(owner, layer, key, value, salience, parent)
23//!   │
24//!   ├─ Governance:  ImportanceScorer::new(salience) + on_update(+5)
25//!   │               TierStateMachine::evaluate → Draft / Validated / Core
26//!   │
27//!   ├─ Storage:     BlockPool::append → Q4-compress key+value, fsync to segment
28//!   │               (on-disk metadata includes importance + tier for crash recovery)
29//!   │
30//!   ├─ Retrieval:   BruteForceRetriever::insert → index key for latent-space queries
31//!   │               SLB::insert → cache key in INT8 for sub-5μs hot-path lookups
32//!   │
33//!   ├─ Index:       if Vamana active → insert_online (graph ANN)
34//!   │               if cell_count ≥ threshold → lazy-activate Vamana
35//!   │
36//!   └─ Trace:       if parent_cell_id provided →
37//!                     WAL::append(AddEdge) → fsync (Observer pattern)
38//!                     TraceGraph::add_edge(child → parent, CausedBy)
39//! ```
40//!
41//! Governance is computed **before** persistence so the on-disk metadata matches
42//! the in-memory state — this is required for correct Memento-pattern recovery
43//! when the engine reopens.
44//!
45//! # What happens when you read
46//!
47//! A single call to [`mem_read`] runs a three-stage retrieval chain
48//! (**Chain of Responsibility** pattern), merging results across stages:
49//!
50//! ```text
51//! mem_read(query_key, k, owner_filter)
52//!   │
53//!   ├─ Stage 1: SLB (hot path)
54//!   │  INT8 quantized keys, NEON SDOT / AVX2
55//!   │  Sub-5μs per query at 4096 entries
56//!   │  Exploits conversational locality — recent cells are cached
57//!   │
58//!   ├─ Stage 2: Vamana (warm path, if active)
59//!   │  DiskANN-style graph with robust pruning
60//!   │  Multi-seed greedy beam search
61//!   │  Activated when cell count crosses threshold
62//!   │
63//!   └─ Stage 3: BruteForce (cold path)
64//!      Exhaustive scaled dot-product: score = (q · k) / √d_k
65//!      Exact results, owner-filtered per-agent partitions
66//!
67//!   Then for each result:
68//!     → recency_decay(Δt) multiplier (21-day half-life)
69//!     → BlockPool::get → dequantize full cell from disk
70//!     → ImportanceScorer::on_access(+3) → potential tier promotion
71//!     → SLB::insert → cache for future hot-path hits
72//! ```
73//!
74//! Results are deduplicated by `CellId`, sorted by decay-adjusted score, and
75//! truncated to k.
76//!
77//! # Crash recovery (Memento pattern)
78//!
79//! On [`Engine::open`], the engine rebuilds all in-memory derived state from
80//! durable sources:
81//!
82//! ```text
83//! Engine::open(dir)
84//!   ├─ BlockPool::open → scan segments, rebuild CellId → offset index
85//!   ├─ For each persisted cell:
86//!   │    BruteForceRetriever::insert(key)
87//!   │    ImportanceScorer::new(persisted_importance)
88//!   │    TierStateMachine::with_tier(persisted_tier)
89//!   ├─ WAL::replay → rebuild TraceGraph from logged edges
90//!   └─ next_id = max(persisted_ids) + 1
91//! ```
92//!
93//! This means the engine can crash at any point and recover all state that was
94//! durably committed (fsync'd). In-flight writes that didn't complete fsync
95//! are discarded — the partial record detection in segment scanning handles this.
96//!
97//! # Causal memory (Trace graph)
98//!
99//! When a cell is written with `parent_cell_id = Some(parent)`, the engine
100//! records a directed `CausedBy` edge in the Trace graph, durably logged via
101//! the WAL before being applied in memory. This enables causal chain queries:
102//!
103//! ```text
104//! trace_ancestors(cell_c) → [cell_b, cell_a]
105//! ```
106//!
107//! If cell C was caused by B, and B was caused by A, the engine traverses
108//! the causal graph transitively.
109//!
110//! # Synaptic bank (`LoRA` adapters)
111//!
112//! Beyond episodic KV memory, the engine also persists per-agent **`LoRA` adapters**
113//! via [`store_synapsis`] / [`load_synapsis`]. These encode stable preferences and
114//! patterns as low-rank weight deltas applied to the base model at runtime —
115//! the "slow memory" complement to the fast episodic KV cache.
116//!
117//! # Python bridge
118//!
119//! The `tdb-python` crate wraps this engine via `PyO3`. From Python:
120//!
121//! ```python
122//! import tardigrade_db as tdb
123//! engine = tdb.Engine("/tmp/tardigrade")
124//! cell_id = engine.mem_write(owner=42, layer=12, key=np_key, value=np_val, salience=80.0, parent_cell_id=None)
125//! results = engine.mem_read(query_key=np_query, k=5, owner=None)
126//! ancestors = engine.trace_ancestors(cell_id)
127//! ```
128//!
129//! # Usage (Rust)
130//!
131//! ```rust,no_run
132//! use tdb_engine::engine::Engine;
133//!
134//! let dir = std::path::Path::new("/tmp/tardigrade");
135//! let mut engine = Engine::open(dir).unwrap();
136//!
137//! // Capture a KV pair from transformer layer 12.
138//! let cell_id = engine.mem_write(
139//!     /* owner */ 42,
140//!     /* layer */ 12,
141//!     /* key */ &[0.1f32; 128],
142//!     /* value */ vec![0.2f32; 128],
143//!     /* salience */ 60.0,
144//!     /* parent */ None,
145//! ).unwrap();
146//!
147//! // Retrieve the most relevant cells for a query key.
148//! let results = engine.mem_read(&[0.1f32; 128], /* k */ 5, None).unwrap();
149//! if let Some(top) = results.first() {
150//!     println!("cell {} score={:.3} tier={:?}", top.cell.id, top.score, top.tier);
151//! }
152//!
153//! // Query causal ancestors.
154//! let ancestors = engine.trace_ancestors(cell_id);
155//!
156//! // Close and reopen — all state is recovered from disk.
157//! drop(engine);
158//! let engine2 = Engine::open(dir).unwrap();
159//! assert_eq!(engine2.cell_count(), 1);
160//! ```
