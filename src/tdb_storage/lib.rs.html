1//! Persistent quantized storage for KV-cache tensors — the storage layer of `TardigradeDB`.
2//!
3//! `tdb-storage` owns the write path: it receives [`MemoryCell`] values, compresses
4//! their key and value vectors to Q4 (4-bit), and appends them to an mmap-backed
5//! segment file. Reads dequantize on-the-fly. The entire layer is crash-safe via
6//! append-only writes and a monotone index rebuild on open.
7//!
8//! # Architecture
9//!
10//! ```text
11//! ┌─────────────────────────────────────────────────────┐
12//! │                     BlockPool                        │
13//! │                                                      │
14//! │  in-memory:  BTreeMap<CellId → (segment_id, offset)>│
15//! │                         │                            │
16//! │  on-disk:   ┌──────────┐ ┌──────────┐ ┌──────────┐  │
17//! │             │ seg_000  │ │ seg_001  │ │ seg_002  │  │  ← append-only files
18//! │             │  256 MB  │ │  256 MB  │ │ (active) │  │
19//! │             └──────────┘ └──────────┘ └──────────┘  │
20//! └─────────────────────────────────────────────────────┘
21//! ```
22//!
23//! Each segment file is a sequential log of binary records. New records are always
24//! appended; existing records are never mutated. On open, [`BlockPool`] scans all
25//! segments to rebuild the `CellId → location` index — this is the recovery path.
26//!
27//! # Quantization
28//!
29//! Key and value vectors are stored in **Q4 group-wise 4-bit** format, matching the
30//! `Q4_0` scheme from [llama.cpp]. Each group of 32 floats shares one `f32` scale
31//! factor; the 32 values are quantized to 4-bit unsigned integers and packed two-per-byte.
32//!
33//! Compression characteristics vs. raw `f32`:
34//! - **Size**: ~4× smaller (4 bits vs 32 bits per element, plus scale overhead)
35//! - **Error**: mean squared error < 0.01 on typical activation distributions
36//! - **Speed**: dequantize is a simple multiply-and-shift, negligible vs. I/O
37//!
38//! Position encodings are stored **unquantized** (`f32`) because they must be
39//! reproduced exactly for safe historical KV block reuse.
40//!
41//! # Segment File Format
42//!
43//! ```text
44//! ┌─────────────────────────┐
45//! │ magic: "TDBS" (4 bytes) │
46//! │ version: u32 (4 bytes)  │
47//! ├─────────────────────────┤  ← FILE_HEADER_SIZE = 8
48//! │ record_len: u32         │  ← total bytes of this record (excludes this field)
49//! │ cell_id: u64            │
50//! │ owner: u64              │
51//! │ layer: u16              │
52//! │ … fixed metadata …      │
53//! │ key (Q4 packed)         │
54//! │ value (Q4 packed)       │
55//! │ pos_encoding (f32[])    │
56//! ├─────────────────────────┤
57//! │ … next record …         │
58//! └─────────────────────────┘
59//! ```
60//!
61//! All integers are **little-endian**. The `record_len` prefix enables the recovery
62//! scanner to skip partial records at EOF without corrupting the index.
63//!
64//! # Recovery
65//!
66//! [`block_pool::BlockPool::open`] rebuilds the entire in-memory index by calling [`scan_segment`]
67//! on every segment file. `scan_segment` reads only the `record_len` and `cell_id`
68//! prefix of each record — it does not deserialize the full tensor — making recovery
69//! proportional to the number of cells, not their size. Partial records at EOF
70//! (from a crash mid-write) are silently discarded.
71//!
72//! # Usage
73//!
74//! ```rust,no_run
75//! use tdb_storage::block_pool::BlockPool;
76//! use tdb_core::memory_cell::MemoryCellBuilder;
77//!
78//! let dir = std::path::Path::new("/tmp/tdb-data");
79//! let mut pool = BlockPool::open(dir).unwrap();
80//!
81//! // Write a cell (key/value are compressed to Q4 on disk).
82//! let cell = MemoryCellBuilder::new(0, 1, 12, vec![0.5f32; 64], vec![0.3f32; 64]).build();
83//! let cell_id = pool.append(&cell).unwrap();
84//!
85//! // Read it back (dequantized from Q4).
86//! let retrieved = pool.get(cell_id).unwrap();
87//! assert_eq!(retrieved.id, 0);
88//! ```
89//!
90//! [`MemoryCell`]: tdb_core::memory_cell::MemoryCell
91//! [`BlockPool`]: block_pool::BlockPool
92//! [`scan_segment`]: segment::scan_segment
93//! [llama.cpp]: https://github.com/ggerganov/llama.cpp
