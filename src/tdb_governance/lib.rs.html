1//! Adaptive Knowledge Lifecycle (AKL) — autonomous memory management for `TardigradeDB`.
2//!
3//! `tdb-governance` implements the self-curating layer of the Aeon architecture.
4//! It answers the question: *given unlimited memory cells accumulating over time,
5//! which ones should be kept, promoted, demoted, or evicted — and when?*
6//!
7//! No application code manages cell lifecycle. The AKL does it automatically based
8//! on observed access patterns and elapsed time.
9//!
10//! # Importance Score (ι)
11//!
12//! Every cell carries an importance score ι ∈ \[0.0, 100.0\] managed by [`ImportanceScorer`].
13//!
14//! ```text
15//! Event        │ Effect on ι
16//! ─────────────┼──────────────────────────────────
17//! Read access  │ +3.0  (capped at 100.0)
18//! Write/update │ +5.0  (capped at 100.0)
19//! Each day     │ × 0.995  (≈ 0.5% decay per day)
20//! ```
21//!
22//! The daily decay factor of 0.995 produces a **half-life of approximately 138 days**
23//! for a cell that is never accessed (ln(0.5) / ln(0.995) ≈ 138). A cell accessed
24//! daily stabilizes around ι ≈ 60 (3 / (1 - 0.995)).
25//!
26//! # Tier State Machine
27//!
28//! [`TierStateMachine`] maps ι to one of three maturity tiers, using **hysteresis
29//! gaps** to prevent oscillation when scores hover near a boundary:
30//!
31//! ```text
32//! ┌─────────────────────────────────────────────────────────────────┐
33//! │                                                                 │
34//! │  DRAFT ──────(ι ≥ 65)──────► VALIDATED ──────(ι ≥ 85)──────► CORE
35//! │         ◄────(ι < 35)────                ◄────(ι < 60)────      │
36//! │                                                                 │
37//! │  Hysteresis gaps:  Draft↔Validated = 30 pts                    │
38//! │                    Validated↔Core  = 25 pts                    │
39//! └─────────────────────────────────────────────────────────────────┘
40//! ```
41//!
42//! Skipping tiers is supported: a new cell with ι = 90 will transition
43//! `Draft → Validated → Core` in a single [`TierStateMachine::evaluate`] call.
44//!
45//! # Recency Decay
46//!
47//! [`recency_decay`] computes an exponential multiplier applied during retrieval
48//! to down-weight cells that haven't been touched recently:
49//!
50//! ```text
51//! r = exp(-Δt / τ),  τ = 30 days  (half-life ≈ 20.8 days)
52//!
53//! Δt = 0 days  →  r = 1.00  (fresh)
54//! Δt ≈ 21 days →  r ≈ 0.50  (half-weight)
55//! Δt = 30 days →  r ≈ 0.37  (one time-constant)
56//! Δt = 90 days →  r ≈ 0.05  (near-zero)
57//! ```
58//!
59//! This is applied as a **score multiplier** at query time, not as a modification
60//! to the stored ι. It means a stale cell with a high importance score will still
61//! rank lower than a recently-accessed cell with a moderate score.
62//!
63//! # Usage
64//!
65//! ```rust
66//! use tdb_governance::scoring::ImportanceScorer;
67//! use tdb_governance::tiers::TierStateMachine;
68//! use tdb_governance::decay::recency_decay;
69//! use tdb_core::Tier;
70//!
71//! let mut scorer = ImportanceScorer::new(50.0);
72//! scorer.on_access();   // +3 → 53.0
73//! scorer.on_update();   // +5 → 58.0
74//! scorer.apply_daily_decay(30); // ×0.995^30 ≈ ×0.861 → ~49.9
75//!
76//! let mut tier = TierStateMachine::new(); // starts at Draft
77//! tier.evaluate(scorer.importance());    // still Draft (< 65)
78//!
79//! // Recency factor for a cell last updated 21 days ago.
80//! let r = recency_decay(21.0);
81//! assert!(r > 0.49 && r < 0.51); // ≈ half-life
82//!
83//! // Final adjusted retrieval score:
84//! let adjusted = scorer.importance() * r;
85//! ```
