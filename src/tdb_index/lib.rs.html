1//! Organization layer for `TardigradeDB`: ANN graph, causal graph, and write-ahead log.
2//!
3//! `tdb-index` implements the **neuro-symbolic dual topology** described in the Aeon
4//! architecture: two complementary data structures that organize memory cells, plus a
5//! WAL for crash recovery of the causal graph.
6//!
7//! # Vamana Graph — Vector Index
8//!
9//! [`VamanaIndex`] is a **DiskANN-style single-layer graph** for approximate nearest
10//! neighbor (ANN) search. It is the cold-path index used when the per-agent cell count
11//! grows beyond the brute-force threshold (~10K) handled by `tdb-retrieval`.
12//!
13//! ```text
14//! Insert phase:                         Query phase:
15//! ┌───────┐  insert(id, vec)            query(q, k=5)
16//! │       │ ──────────────────►         │
17//! │ nodes │   add node to list          ▼ greedy beam search
18//! │       │                        seed at medoid
19//! └───────┘  build()               expand best candidates
20//!             │                    explore neighbor edges
21//!             ▼                    return top-k by score
22//!    connect each node to
23//!    its R nearest neighbors
24//! ```
25//!
26//! **Key design choices vs. HNSW:**
27//! - Single layer (no multi-layer hierarchy) → simpler build, faster cold-start
28//! - Medoid seeding → avoids pathological cases when queries are far from the centroid
29//! - Page-clustered insertions (future) → `DiskANN`'s disk-locality trick for `NVMe` I/O
30//!
31//! Configuration:
32//! - `max_degree` (R): out-degree per node. Higher = better recall, more memory.
33//! - `search_list_size` (L) is set automatically to `max(k, R) * 2` during queries.
34//!
35//! # Trace Graph — Causal Index
36//!
37//! [`TraceGraph`] is a **directed episodic graph** where nodes are [`CellId`]s and
38//! edges represent causal relationships between stored memories:
39//!
40//! ```text
41//!   cell_A ──[CausedBy]──► cell_B ──[CausedBy]──► cell_C
42//!   cell_A ──[Supports]──► cell_D
43//!   cell_E ──[Contradicts]► cell_A
44//! ```
45//!
46//! Edge types: `CausedBy`, `Follows`, `Supports`, `Contradicts`.
47//!
48//! The graph supports transitive ancestor queries (`ancestors(node, EdgeType)`),
49//! which an agent can use to reconstruct the causal chain that led to any stored memory.
50//!
51//! # Write-Ahead Log — Crash Recovery
52//!
53//! [`Wal`] is an append-only binary log that records every [`TraceGraph`] mutation
54//! before it is applied. On restart, mutations are replayed from the WAL before the
55//! in-memory graph is used.
56//!
57//! ```text
58//! Graph mutation requested
59//!        │
60//!        ▼
61//!   WAL.append(entry)   ← fsync to disk FIRST
62//!        │
63//!        ▼
64//!   TraceGraph.add_edge(...)   ← then apply in memory
65//!        │
66//!        ▼
67//!   (periodic) WAL.checkpoint()  ← truncate WAL after snapshot
68//! ```
69//!
70//! WAL record format (little-endian): `type(1) | src(8) | dst(8) | edge_type(1) | timestamp(8)`
71//! = 26 bytes per record. Fixed-size records allow O(1) position arithmetic during replay.
72//!
73//! # Usage
74//!
75//! ```rust,no_run
76//! use tdb_index::vamana::VamanaIndex;
77//! use tdb_index::trace::{TraceGraph, EdgeType};
78//! use tdb_index::wal::Wal;
79//!
80//! // Build a vector index for cold-path ANN retrieval.
81//! let mut idx = VamanaIndex::new(/* dim */ 128, /* max_degree */ 32);
82//! idx.insert(0, &[0.5f32; 128]);
83//! idx.insert(1, &[0.1f32; 128]);
84//! idx.build(); // O(n²) — call after all inserts
85//! let results = idx.query(&[0.5f32; 128], /* k */ 1);
86//! assert_eq!(results[0].0, 0);
87//!
88//! // Record a causal edge between two cells (WAL-backed).
89//! let dir = std::path::Path::new("/tmp/tdb-wal");
90//! let mut wal = Wal::open(dir).unwrap();
91//! let mut graph = TraceGraph::new();
92//! let entry = tdb_index::wal::WalEntry::AddEdge {
93//!     src: 0, dst: 1, edge_type: EdgeType::CausedBy as u8, timestamp: 1000,
94//! };
95//! wal.append(&entry).unwrap();        // durable FIRST
96//! graph.add_edge(0, 1, EdgeType::CausedBy, 1000); // then in memory
97//! ```
