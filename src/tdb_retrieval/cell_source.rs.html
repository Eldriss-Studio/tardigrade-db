1//! `CellSource` — the storage seam for lazy retrievers.
2//!
3//! Retrievers that want to score cell data on demand (rather than holding
4//! a full in-memory copy) depend on this trait instead of `BlockPool`
5//! directly. The trait is intentionally minimal: one operation, one
6//! return type, no error channel beyond `Option` for "not found." Bigger
7//! surface would tie retrievers to storage semantics (segments, fsync,
8//! recovery) they have no business knowing.
9//!
10//! # Ownership model
11//!
12//! Sources are passed **by borrowed reference** to retriever and pipeline
13//! query methods. The retriever does not store the source; it is borrowed
14//! for the duration of one query. This keeps `BlockPool` ownership in
15//! the engine and avoids `Arc<RwLock<BlockPool>>` plumbing through the
16//! retrieval crate.
17//!
18//! # Design pattern
19//!
20//! Strategy: callers depend on the trait; concrete storage backends
21//! provide the impl. Test doubles implement the trait directly without
22//! any storage layer.
23//!
24//! # Example test double
25//!
26//! ```
27//! use std::collections::HashMap;
28//! use tdb_core::CellId;
29//! use tdb_retrieval::cell_source::CellSource;
30//!
31//! struct InMemoryCellSource {
32//!     cells: HashMap<CellId, Vec<f32>>,
33//! }
34//!
35//! impl CellSource for InMemoryCellSource {
36//!     fn get_encoded_key(&self, id: CellId) -> Option<Vec<f32>> {
37//!         self.cells.get(&id).cloned()
38//!     }
39//! }
40//! ```

44/// Storage seam for retrievers that score cells on demand.
45///
46/// Implementations must be safe to call from multiple threads — the
47/// retriever holds an `Arc<dyn CellSource>` and may dispatch from
48/// concurrent query contexts.
49pub trait CellSource: Send + Sync {
50    /// Return a clone of the cell's encoded retrieval key.
51    ///
52    /// Returns `None` if the cell does not exist (e.g., deleted,
53    /// never written, or `id` outside the source's domain). Callers
54    /// must treat `None` as "skip this candidate," not as an error.
55    fn get_encoded_key(&self, id: CellId) -> Option<Vec<f32>>;
56}
