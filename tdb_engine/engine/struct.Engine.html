pub fn mem_write_batch( &mut self, requests: &[WriteRequest], ) -> Result<Vec<CellId>>

Write multiple cells in a single batch with one fsync (Batch Command pattern).

All cells are persisted to the block pool in one operation, then indexed into the retrieval pipeline, SLB, and governance. Causal edges (if any) are written to the WAL after the batch is persisted.

Throughput: ~80us/cell amortized vs ~8ms/cell for individual writes.

Source

pub fn mem_read( &mut self, query_key: &[f32], k: usize, owner_filter: Option<OwnerId>, ) -> Result<Vec<ReadResult>>

Read the top-k most relevant cells for a query key.

Encoded per-token queries run the [RetrieverPipeline] first so PerTokenRetriever(Top5Avg) scores the raw token matrix. The SLB uses mean-pooled summaries only as a fallback/hot cache because it requires fixed-size vectors. Results are merged, deduplicated, scored with recency decay, and filtered by owner.
