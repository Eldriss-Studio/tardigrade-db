Append-only deletion log — durable persistence for pack deletions.

Records PackId values that have been deleted. On open, the file is replayed into a HashSet which serves as a negative filter over the pack directory.

§File format

┌────────────────────────────┐
│ pack_id: u64 (8 bytes, LE) │
├────────────────────────────┤
│ … next record …            │
└────────────────────────────┘

Each record is exactly 8 bytes. Truncated trailing bytes (< 8) are silently discarded.

§Durability contract

DeletionLog::mark_deleted appends and fsyncs before returning. A crash after mark_deleted returns guarantees the deletion is on disk.

§Recovery contract

DeletionLog::open rebuilds the HashSet from the file. On engine open, the deleted set is applied as a filter to the pack directory.
